# Flay-Ground — Claude 지침

영화 콘텐츠 관리 및 스트리밍 웹 애플리케이션. Node.js 풀스택 프로젝트.

## 프로젝트 구조

```
flayground/
├── web-backend/     # Node.js + Express REST API (TypeScript, tsx/tsup)
├── web-frontend/    # Webpack + TypeScript 멀티 엔트리 SPA (Web Components)
├── mcp-nexus/       # MCP AI 라우터 (Gemini / GitHub Models)
├── playwright/      # E2E 테스트
├── bin/             # 실행 스크립트
└── doc/             # 문서
```

## 기본 규칙

- **응답 언어**: 모든 답변·주석·커밋 메시지 제안은 한국어로 작성
- **패키지 매니저**: 모든 Node.js 프로젝트에서 `npm` 대신 `yarn` 사용
- **JSDoc**: 모든 함수·클래스·인터페이스에 파라미터 타입·반환 타입·목적을 포함한 JSDoc 작성
- 복잡한 로직은 인라인 주석으로 최소한 보강

## 코드 원칙 (우선순위 순)

**① 단순함 > ② 변경 범위 최소화 > ③ 함수 응집도**

규칙이 충돌하면 우선순위가 높은 쪽을 따른다.

### 단순함 우선

- 요청하지 않은 기능 추가 금지
- 단일 사용 코드에 추상화 계층 금지
- 발생 불가능한 시나리오에 대한 오류 처리 금지
- 200줄로 쓴 코드가 50줄로 가능하다면 다시 작성

### 변경 범위 최소화

- 요청된 것만 수정; 인접 코드·포맷을 임의로 "개선" 금지
- 깨지지 않은 것은 리팩토링 금지
- 기존 스타일을 따를 것
- 관련 없는 데드 코드 발견 시 언급만 할 것 (삭제 금지)
- 자신의 변경으로 불필요해진 import/변수/함수는 제거

### 함수 분리 기준

| 분리 권장 | 분리 지양 |
|-----------|-----------|
| 재사용 가능 로직 | 한 곳에서만 쓰는 단순 계산 |
| 50줄 이상 복잡 로직 | 5줄 이하 유틸리티 |
| 독립 테스트 필요 | 단순 템플릿 |

- 호출 깊이 3단계 초과 지양
- 코드 점프 최소화 (한 흐름에서 맥락 파악 가능하게)

## 구현 전 체크

암묵적 가정을 하지 말 것:

- 여러 해석이 가능하면 선택지를 제시 (조용히 하나를 선택하지 말 것)
- 더 단순한 접근이 있다면 명시적으로 제안
- 불분명한 것이 있으면 멈추고 질문

## Playwright 테스트

- 이전 브라우저가 닫혀 있을 가능성이 높음 → 새 브라우저를 열어 테스트
- 웹팩 dev 서버(`yarn dev`)가 실행 중인지 먼저 확인
- **URL 기준**: `https://flay.kamoru.jk/dist/` + HTML 파일명 (예: `page.history-shot.html`)
- 테마: `<html theme="dark">` 다크 테마로 실행됨
- 스크린샷 저장: `filename`은 항상 `.playwright-mcp/` 하위로 지정, `type`과 확장자 반드시 일치
